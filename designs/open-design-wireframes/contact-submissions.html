Contact Submissions

Admin inbox for public contact form messages with statuses, filters, detail panel, and archive flow.

①

Admin nav shows new submission count and keeps inbox separate from moderation queues.

②

Search/filter by status, subject, sender email, and received date.

③

Inbox table pairs with detail panel to avoid route churn while triaging messages.

④

Statuses: new, read, archived. Reply path should preserve original message context.

⑤

Loading, empty inbox, archive success, and error states are required before MVP launch.

Pin selection is interactive for review.
ResponsiveDesktop split inbox/detail; mobile detail appears after selected message.
A11yStatus filters need selected states; message rows need descriptive labels and focus restoration.
StateArchive and read changes should show confirmation and recover from failed requests.
