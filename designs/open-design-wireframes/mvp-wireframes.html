MVP route overview

Launcher for the Babas & Brasse MVP wireframe set. Home remains the existing `index.html` route.

①

This overview is a project browser, not a product screen; route links use normal anchors for quick review.

②

MVP summary keeps the launch target visible: responsive web app wireframes for July 31, 2026.

③

Public routes include home, about, team, contributors, archive, article, search, media, and contact.

④

Admin routes cover dashboard, articles, profiles/media, moderation, and contact submissions.

⑤

Support wireframes cover auth, password reset, 404, 500, offline/maintenance, media upload, article editor workflow, and mobile-only comps.

⑥

Implementation should map each file to a route and preserve public/admin/support access boundaries.

Pin selection is interactive for review.
RoutingEach listed HTML file is a separate screen-first route wireframe.
A11yLauncher links are keyboard-focusable and keep visible focus treatment.
BoundaryPublic pages show only published/approved content; admin pages require role checks; support screens handle recovery and edge states.
