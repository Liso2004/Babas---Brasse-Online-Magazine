Admin Dashboard wireframe

Operational overview for editors managing articles, comments, reviews, media, and contact submissions.

①

Admin nav is separate from public nav and requires authenticated editor access plus visible current route.

②

Overview stats: published articles, drafts, pending comments, pending reviews, and new contact submissions.

③

Recent activity table shows actor, item, status, timestamp, and next action for editorial operations.

④

Quick actions link to create article, moderation, media upload, and contact inbox.

⑤

Admin surfaces need loading, empty, error, and permission-denied states with retry or sign-in route.

Pin selection is interactive for review.
ResponsiveDesktop dense grid; tablet two-column metrics; mobile admin remains usable as stacked panels.
A11yTables need headers, row actions, keyboard focus, and status text not color-only.
