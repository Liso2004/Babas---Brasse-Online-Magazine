Comments/Reviews Moderation

Queue-driven moderation with pending, approved, rejected, search, filter, and approve/reject/delete actions.

①

Admin nav separates moderation from article editing and should show pending counts.

②

Queues: pending, approved, rejected, plus search and filters across article, author, type, and date.

③

Dense queue table with a selected-item detail panel so editors can review context before acting.

④

Approve, reject, delete, search, and filter actions must be keyboard reachable and confirmation-aware for delete.

⑤

Public article pages only show approved comments/reviews; rejected and pending content never renders publicly.

Pin selection is interactive for review.
ResponsiveDesktop split queue/detail; mobile detail opens below selected row.
A11yQueue tabs need selected state, buttons need labels, destructive actions need confirmation.
StateOptimistic moderation should offer undo and show server failure feedback.
