Admin Login wireframe

Admin authentication, recovery path, role gate, and security states.

①

Login is an admin support route, separate from public navigation and clearly branded as B&B Admin.

②

Form requires email, password, submit button, labelled fields, visible errors, and disabled state while pending.

③

States cover invalid credentials, locked account, loading, and successful redirect to admin dashboard.

④

Recovery links include forgot password and return-to-public-site paths without exposing admin internals.

⑤

Security handoff: role check after auth, session expiry, CSRF protection, and audit logging for admin access.
