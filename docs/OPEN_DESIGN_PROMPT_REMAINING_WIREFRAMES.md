# Open Design Prompt For Remaining MVP Wireframes

Use this prompt in Open Design to create the remaining Babas & Brasse MVP wireframes.

```text
Create annotated low-fidelity wireframes for the remaining Babas & Brasse Online Magazine MVP screens. Match the existing annotated landing/home wireframe style from the current project: grayscale redline, B&B mark, mono annotation labels, numbered pins, right-side spec panel, accessible button pins, and practical engineering notes. Do not make polished final visuals yet.

Context:
- Project: Babas & Brasse Online Magazine MVP.
- Launch deadline: July 31, 2026.
- Existing Home/Landing wireframe already covers sticky nav, search, subscribe CTA, lead theatre story, section rail, recent article teasers, newsletter/footer, and annotation panel.
- Keep visual hierarchy consistent with that Home/Landing artifact.
- Design for a responsive web app, not a native mobile app.
- Use route/screen-first thinking so implementation can map each screen to a route.

Required public screens to wireframe:
1. About
   - Mission, Vision, Organisational Overview.
   - Link paths to Creative Team, Contributors, Contact.
   - Annotation notes for content hierarchy and mobile stacking.

2. Creative Team
   - Team profile grid/list.
   - Profile card anatomy: image, name, role, short bio, social links.
   - Empty/loading state note if team content is missing.

3. Contributors
   - Contributor profile grid/list.
   - Published works links per contributor.
   - Filter/search affordance if contributor count grows.

4. Visceral Mag article archive
   - Article listing with category filters and search.
   - Published-only public article cards.
   - Metadata: category, title, dek, author, date, read time.
   - Empty/no-results and search-loading states.

5. Article Detail
   - Article hero with category, title, dek, author, date, read time, featured image.
   - Body content rhythm.
   - Related articles.
   - Approved-only comments.
   - Approved-only reviews/rating area.
   - SEO metadata handoff note.

6. Categories/Search
   - Search input, category chips, result list, filters.
   - Loading, no-results, error, and reset states.

7. Featured/Media
   - Photography/artwork/media grid.
   - Media card anatomy: image, title, caption, credit, alt text note.
   - Image-heavy performance and responsive notes.

8. Contact
   - Contact form with name, email, subject, message.
   - Spam protection placeholder.
   - Validation, success, and error states.
   - Contact info/social links area.

Required admin screens to wireframe:
9. Admin Dashboard
   - Overview stats: published articles, drafts, pending comments, pending reviews, new contact submissions.
   - Recent activity.
   - Quick actions.

10. Article Management
   - Article table with title, status, category, author, date, SEO readiness.
   - Draft/publish states.
   - Create/edit form anatomy: title, slug, dek, body editor, category, author, featured image, alt text, SEO title, SEO description, Open Graph fields.

11. Profile/Media Management
   - Contributor and Creative Team management.
   - Media library with image metadata: alt text, caption, credit, category.
   - Upload/select placeholder.

12. Comments/Reviews Moderation
   - Pending, approved, rejected queues.
   - Approve, reject, delete, search, filter.
   - Note that public pages only show approved comments/reviews.

13. Contact Submissions
   - Inbox table.
   - Detail panel.
   - Statuses: new, read, archived.
   - Search/filter.

For every screen:
- Include numbered annotation pins and a right-side spec panel.
- Preserve accessibility notes: focus states, keyboard behavior, labels, aria where needed.
- Include responsive notes for desktop/tablet/mobile.
- Include state notes: loading, empty, error, success where relevant.
- Avoid generic marketing sections. Make it feel like a practical editorial magazine and publishing admin tool.
- Keep admin screens dense, operational, and table/form driven.
- Do not introduce final brand colors or decorative gradients; stay in annotated redline mode.
```
